# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за проект

Реализация **MCP (Model Context Protocol) сервера** внутри расширения 1С:Предприятие 8.3. Позволяет AI-клиентам (Claude и другим) взаимодействовать с 1С-системой через стандартные MCP-эндпоинты — запрашивать инструменты (tools), ресурсы (resources) и промпты (prompts).

Автор: Vladimir Kharin (C) 2025. Источник: https://github.com/vladimir-kharin/1c_mcp

## Запуск и тестирование

Это расширение 1С, не Node.js/Python-проект — традиционных команд build/test нет.

- **Деплой:** установить расширение в конфигурацию 1С, HTTP-сервис поднимается на базовом пути `/mcp`
- **Health check:** `GET /mcp/health`
- **Тесты:** открыть обработку `DataProcessors/Тесты` в 1С и запустить форму — там 100+ тест-кейсов для HTTP-коннектора
- **Прямое выполнение:** `POST /mcp/execute` (произвольный BSL-код), `POST /mcp/query` (запрос к БД)

## Архитектура

### Точка входа

`HTTPServices/mcp_APIBackend/Ext/Module.bsl` — HTTP-сервис. Маршруты:

| Метод | Путь | Назначение |
|-------|------|-----------|
| POST | `/mcp/rpc` | JSON-RPC (legacy) |
| POST | `/mcp/*` | MCP JSON-RPC (основной) |
| GET  | `/mcp/health` | Health check |
| POST | `/mcp/execute` | Выполнение BSL-кода |
| POST | `/mcp/query` | Выполнение запроса |

Все запросы обрабатывает `ОбработатьJSONRPCЗапрос()`, реализующий JSON-RPC 2.0 (версия протокола MCP `2025-03-26`).

### Плагинная архитектура (Container Pattern)

Инструменты/ресурсы/промпты регистрируются через **подсистемы-контейнеры**:

```
Подсистема mcp_КонтейнерыИнструментов
    └── DataProcessors (каждый реализует ДобавитьИнструменты() и ВыполнитьИнструмент())

Подсистема mcp_КонтейнерыРесурсов
    └── DataProcessors (каждый реализует ДобавитьРесурсы() и ПрочитатьРесурс())

Подсистема mcp_КонтейнерыПромптов
    └── DataProcessors (каждый реализует ДобавитьПромпты() и ПолучитьПромпт())
```

Обнаружение при старте: `mcp_КонтейнерыПовтИсп` (общий модуль с повторным использованием) кешируют таблицы через `mcp_Метаданные`, который обходит все объекты нужной подсистемы.

### Поток вызова инструмента

```
POST /mcp  →  ОбработатьJSONRPCЗапрос()
    → "tools/call"  →  mcp_Выполнение.ВыполнитьИнструмент()
        → находит строку в кешированной таблице по имени инструмента
        → ОбработчикМенеджера().ВыполнитьИнструмент(Имя, Аргументы)
            → возвращает массив объектов mcp_Содержимое
```

### Ключевые общие модули

| Модуль | Роль |
|--------|------|
| `mcp_ОбщегоНазначения` | JSON↔Структура, парсинг URL, нормализация ключей |
| `mcp_Метаданные` | Реестры инструментов/ресурсов/промптов, построение JSON Schema |
| `mcp_Выполнение` | Диспетчер вызовов к контейнерам |
| `mcp_Содержимое` | Фабрика MCP content-объектов (text, image, binary, message) |
| `mcp_КонтейнерыПовтИсп` | Кеш таблиц инструментов/ресурсов/промптов |
| `КоннекторHTTP` | HTTP-клиент (внешняя библиотека) |

### Существующие контейнеры (DataProcessors)

- `mcp_ИнструментДанныеОКонфигурации` — инструменты интроспекции метаданных конфигурации (структура объектов, реквизиты, ТЧ, измерения регистров)
- `mcp_РесурсОписаниеСинтаксисаВстроенногоЯзыка` — ресурс `file://resource/syntax_1c.txt` с документацией по BSL
- `mcp_УправлениеСервером` — административные инструменты (скелет)
- `Тесты` — тест-кейсы для HTTP-коннектора

## Как добавить функционал в MCP

### Новый инструмент (tool)

Инструмент — функция, которую AI вызывает с параметрами и получает результат.

**Шаг 1.** Создать Обработку с именем вида `mcp_ИнструментXxx` в конфигурации 1С.

**Шаг 2.** В `Subsystems/mcp_MCPСервер/Subsystems/mcp_КонтейнерыИнструментов.xml` добавить строку в `<Content>`:
```xml
<xr:Item xsi:type="xr:MDObjectRef">DataProcessor.mcp_ИнструментXxx</xr:Item>
```

**Шаг 3.** Написать `Ext/ManagerModule.bsl` по шаблону:

```bsl
// Регистрация инструментов — вызывается при tools/list
Процедура ДобавитьИнструменты(Инструменты) Экспорт
    МассивПараметров = Новый Массив;

    // Простой параметр: (Имя, Тип, Описание, ЗначениеПоУмолчанию, Обязательный, ДопустимыеЗначения)
    МассивПараметров.Добавить(mcp_Метаданные.ПараметрИнструмента(
        "paramName", "string", "Описание параметра", , Истина
    ));

    // Параметр-массив: (Имя, ТипЭлемента, Описание, Обязательный)
    // МассивПараметров.Добавить(mcp_Метаданные.ПараметрИнструментаМассив("items", "string", "Список", Ложь));

    mcp_Метаданные.ДобавитьИнструмент(
        Инструменты,
        "tool_name",           // имя инструмента (snake_case, англ.)
        "Описание инструмента",
        mcp_Метаданные.СхемаПараметровИнструмента(МассивПараметров)
    );
КонецПроцедуры

// Выполнение — вызывается при tools/call
Функция ВыполнитьИнструмент(ИмяИнструмента, Аргументы) Экспорт
    Если ИмяИнструмента = "tool_name" Тогда
        Возврат МойИнструмент(Аргументы);
    Иначе
        ВызватьИсключение "Неизвестный инструмент: " + ИмяИнструмента;
    КонецЕсли;
КонецФункции

Функция МойИнструмент(Аргументы)
    Значение = Аргументы.paramName;
    // ... логика ...
    Возврат mcp_Содержимое.ТекстовоеСодержимое("Результат: " + Значение);
КонецФункции
```

**Возврат из инструмента:**
- `mcp_Содержимое.ТекстовоеСодержимое(Строка)` — текстовый ответ
- `mcp_Содержимое.ИзображениеСодержимое(Base64, MimeТип)` — изображение
- Массив из нескольких таких объектов — несколько блоков контента
- `ВызватьИсключение "..."` — ответ с `isError: true` (ошибка перехватывается MCP-слоем)

### Новый ресурс (resource)

Ресурс — статический или динамический контент (файл, документация), который AI читает.

**Шаг 1.** Создать Обработку `mcp_РесурсXxx`, добавить в `mcp_КонтейнерыРесурсов.xml`.

**Шаг 2.** `ManagerModule.bsl`:

```bsl
Процедура ДобавитьРесурсы(ТаблицаРесурсов) Экспорт
    mcp_Метаданные.ДобавитьРесурс(
        ТаблицаРесурсов,
        "file://resource/my_resource",  // URI ресурса
        "my_resource",                  // имя
        "Описание ресурса"
    );
КонецПроцедуры

Функция ПрочитатьРесурс(Адрес) Экспорт
    Возврат mcp_Содержимое.ТекстовоеСодержимоеРесурса(
        "Текст содержимого ресурса...",
        "text/plain"
    );
КонецФункции
```

---

### Новый промпт (prompt)

Промпт — шаблон сообщения с параметрами, который AI использует как заготовку.

**Шаг 1.** Создать Обработку `mcp_ПромптXxx`, добавить в `mcp_КонтейнерыПромптов.xml`.

**Шаг 2.** `ManagerModule.bsl`:

```bsl
Процедура ДобавитьПромпты(ТаблицаПромптов) Экспорт
    МассивПараметров = Новый Массив;
    МассивПараметров.Добавить(mcp_Метаданные.ПараметрПромпта("context", "Контекст задачи", Истина));

    mcp_Метаданные.ДобавитьПромпт(
        ТаблицаПромптов,
        "prompt_name",
        "Описание промпта",
        mcp_Метаданные.ПараметрыПромпта(МассивПараметров)
    );
КонецПроцедуры

Функция ПолучитьПромпт(ИмяПромпта, Параметры) Экспорт
    Текст = "Выполни задачу: " + Параметры.context;
    Сообщения = Новый Массив;
    Сообщения.Добавить(mcp_Содержимое.СообщениеПромпта("user", Текст));
    Возврат Сообщения;
КонецФункции
```

### После любых изменений

1. Обновить расширение в 1С (Конфигуратор → применить расширение)
2. Переподключить MCP-клиент (перезапустить Claude Code), чтобы он запросил актуальный `tools/list`

## Работа с метаданными 1С в коде

При написании BSL-кода всегда проверяй объекты через MCP-инструменты (`mcp__mcp_1c_local__list_metadata_objects` / `mcp__mcp_1c_local__get_metadata_structure`). Не угадывай имена реквизитов.
