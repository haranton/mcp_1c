# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за проект

Реализация **MCP (Model Context Protocol) сервера** внутри расширения 1С:Предприятие 8.3. Позволяет AI-клиентам (Claude и другим) взаимодействовать с 1С-системой через стандартные MCP-эндпоинты — запрашивать инструменты (tools), ресурсы (resources) и промпты (prompts).

Автор: Vladimir Kharin (C) 2025. Источник: https://github.com/vladimir-kharin/1c_mcp

## Запуск и тестирование

Это расширение 1С, не Node.js/Python-проект — традиционных команд build/test нет.

- **Деплой:** установить расширение в конфигурацию 1С, HTTP-сервис поднимается на базовом пути `/mcp`
- **Health check:** `GET /mcp/health`
- **Тесты:** открыть обработку `DataProcessors/Тесты` в 1С и запустить форму — там 100+ тест-кейсов для HTTP-коннектора
- **Прямое выполнение:** `POST /mcp/execute` (произвольный BSL-код), `POST /mcp/query` (запрос к БД)

## Архитектура

### Точка входа

`HTTPServices/mcp_APIBackend/Ext/Module.bsl` — HTTP-сервис. Маршруты:

| Метод | Путь | Назначение |
|-------|------|-----------|
| POST | `/mcp/rpc` | JSON-RPC (legacy) |
| POST | `/mcp/*` | MCP JSON-RPC (основной) |
| GET  | `/mcp/health` | Health check |
| POST | `/mcp/execute` | Выполнение BSL-кода |
| POST | `/mcp/query` | Выполнение запроса |

Все запросы обрабатывает `ОбработатьJSONRPCЗапрос()`, реализующий JSON-RPC 2.0 (версия протокола MCP `2025-03-26`).

### Плагинная архитектура (Container Pattern)

Инструменты/ресурсы/промпты регистрируются через **подсистемы-контейнеры**:

```
Подсистема mcp_КонтейнерыИнструментов
    └── DataProcessors (каждый реализует ДобавитьИнструменты() и ВыполнитьИнструмент())

Подсистема mcp_КонтейнерыРесурсов
    └── DataProcessors (каждый реализует ДобавитьРесурсы() и ПрочитатьРесурс())

Подсистема mcp_КонтейнерыПромптов
    └── DataProcessors (каждый реализует ДобавитьПромпты() и ПолучитьПромпт())
```

Обнаружение при старте: `mcp_КонтейнерыПовтИсп` (общий модуль с повторным использованием) кешируют таблицы через `mcp_Метаданные`, который обходит все объекты нужной подсистемы.

### Поток вызова инструмента

```
POST /mcp  →  ОбработатьJSONRPCЗапрос()
    → "tools/call"  →  mcp_Выполнение.ВыполнитьИнструмент()
        → находит строку в кешированной таблице по имени инструмента
        → ОбработчикМенеджера().ВыполнитьИнструмент(Имя, Аргументы)
            → возвращает массив объектов mcp_Содержимое
```

### Ключевые общие модули

| Модуль | Роль |
|--------|------|
| `mcp_ОбщегоНазначения` | JSON↔Структура, парсинг URL, нормализация ключей |
| `mcp_Метаданные` | Реестры инструментов/ресурсов/промптов, построение JSON Schema |
| `mcp_Выполнение` | Диспетчер вызовов к контейнерам |
| `mcp_Содержимое` | Фабрика MCP content-объектов (text, image, binary, message) |
| `mcp_КонтейнерыПовтИсп` | Кеш таблиц инструментов/ресурсов/промптов |
| `КоннекторHTTP` | HTTP-клиент (внешняя библиотека) |

### Существующие контейнеры (DataProcessors)

- `mcp_ИнструментДанныеОКонфигурации` — инструменты интроспекции метаданных конфигурации (структура объектов, реквизиты, ТЧ, измерения регистров)
- `mcp_РесурсОписаниеСинтаксисаВстроенногоЯзыка` — ресурс `file://resource/syntax_1c.txt` с документацией по BSL
- `mcp_УправлениеСервером` — административные инструменты (скелет)
- `Тесты` — тест-кейсы для HTTP-коннектора

## Как добавить новый инструмент

1. Создать DataProcessor с именем вида `mcp_ИнструментXxx`
2. Включить его в подсистему `mcp_КонтейнерыИнструментов`
3. Реализовать в `ManagerModule.bsl`:
   - `ДобавитьИнструменты(ТаблицаИнструментов)` — регистрация через `mcp_Метаданные.ПараметрИнструмента()` и `mcp_Метаданные.СхемаПараметровИнструмента()`
   - `ВыполнитьИнструмент(Имя, Аргументы)` — логика + возврат через `mcp_Содержимое.ТекстовоеСодержимое()`

## Работа с метаданными 1С в коде

При написании BSL-кода всегда проверяй объекты через MCP-инструменты (`mcp__mcp_1c_local__list_metadata_objects` / `mcp__mcp_1c_local__get_metadata_structure`). Не угадывай имена реквизитов.
